# Codex Per-Prompt Logs

Create one new file in this folder for every user prompt that changes the task, plan, or implementation state.

## Filename

Use Asia/Jakarta local time:

```text
YYYY-MM-DD-HHMM-short-topic.md
```

Example:

```text
2026-06-10-1630-project-rules.md
```

## Required Content

Each log should include:

- Prompt summary.
- Plan or status.
- Files touched or expected.
- Blockers or risks.
- Verification run or skipped.
- Next step.

Keep entries short and factual so the next Codex session can continue quickly.
